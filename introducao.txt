Git - O git é um sistema de versionamento de código, no qual os desenvolvedores podem ter mais flexibilidade em um projeto e mais organização, realizando operações em branches secundárias, como um rascunho, e salvando alterações na branche principal (master ou main). A branche principal é o arquivo que não deve ser alterado. Nele conterá a versão final do código para produção. Todas as alterações são feitas nas branches secundárias (features, etc) e após os testes bem-sucedidos, mesclados na branch master.

Github - Uma comunidade onde um usuário pode criar seus repositórios, públicos ou privados, no qual podem ser salvos projetos versionados, onde usuários que têm permissões podem copiar o projeto (fork) e criar suas próprias funcionalidad contribuindo com a comunidade opensource, ou contribuindo em um projeto pessoal através de fazer pull requests (requisições de implementação de novos recursos criados por algum desenvolvedor). As possoas podem dar likes em projetos, seguir outras pessoas, entre outros. É uma ótima oportunidade para desenvolvedores, pois é uma espécie de portfólio, onde os desenvolvedores criam sua imagem contribuindo com a comunidade em projetos open-source e compartilhando seus melhores projetos.
