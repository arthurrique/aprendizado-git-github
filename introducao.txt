Git - O git é um sistema de versionamento de código, no qual os desenvolvedores podem ter mais flexibilidade e organização em um projeto, realizando operações em branchs secundárias, como um rascunho, e salvando alterações na branch principal (master ou main). A branch principal é o arquivo estático que não deve ser alterado. Nele conterá a versão final do código para produção. Todas as alterações são feitas nas branchs secundárias (chamadas features, etc) e após os testes bem-sucedidos, são mesclados na branch master.

Github - Uma comunidade onde um usuário pode criar seus repositórios, públicos ou privados, no qual os projetos podem ser salvos e versionados, onde usuários que têm permissões podem fazer um fork (copiar o projeto e criar suas próprias funcionalidades contribuindo com a comunidade open-source, ou contribuindo em um projeto privado, através de pull request, que é uma requisição de implementação de novos recursos criados por este desenvolvedor). As possoas podem dar likes em projetos, seguir outras pessoas, entre outros. É uma ótima oportunidade para desenvolvedores, pois é uma espécie de portfólio, onde os desenvolvedores criam sua imagem contribuindo com a comunidade em projetos open-source e compartilhando seus melhores projetos.
